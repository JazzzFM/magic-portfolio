## Overview

[View on GitHub](https://github.com/JazzzFM/AegisFraud)

The cost of a missed fraud signal is measured in milliseconds. AegisFraud (from Greek *aegis*, meaning shield) is a real-time detection platform that intercepts anomalous transactions before they settle—combining stream processing, online model serving, and a dynamic rules engine into a single cohesive system.

## Architecture

The core engineering challenge was bridging two different time horizons: batch model retraining (hours) and real-time inference (milliseconds). AegisFraud solves this with a hybrid pipeline:

- **Ingestion layer**: Apache Kafka absorbs raw transaction streams, decoupling producers from the rest of the system and providing durable replay for recovery scenarios.
- **Real-time inference**: Apache Flink consumes events, extracts features in-flight using stateful operators, and scores each transaction against the active fraud model with sub-second latency.
- **Batch retraining**: Apache Airflow orchestrates nightly retraining jobs that promote updated models through MLflow staging before any change reaches production.
- **Dynamic rules engine**: Fraud analysts can push new detection rules at runtime—without redeploying Flink jobs—allowing the system to respond to emerging attack patterns within minutes.

## Model Lifecycle

MLflow manages the complete model lifecycle: experiment tracking, artifact storage, version promotion, and rollback. A new detection model can move from development to production through a single registry transition, decoupled from infrastructure releases. This separation means the data science cycle and the deployment cycle never block each other.

## Outcome

AegisFraud demonstrates how Kafka, Flink, Airflow, and MLflow—each a best-in-class tool in its domain—can be composed into a system that learns from historical data while acting on live transactions in real time. The architecture is designed to scale horizontally at every layer without changes to application logic.
