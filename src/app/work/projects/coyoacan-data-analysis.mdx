## Overview

[View on GitHub](https://github.com/JazzzFM/CoyoacanDataAnalysis)

Coyoacán is one of Mexico City's most historically and demographically complex boroughs. Urban decision-makers there typically work from siloed datasets—INEGI for demographics, IECM for electoral behavior, local stations for air quality—and correlating them manually is a barrier to data-informed policy. This project removes that barrier by building a unified analytical platform that merges all three into a single queryable system, turning fragmented public records into actionable urban intelligence.

## Technical Implementation

### Geospatial Database

Raw shapefiles from INEGI and IECM were ingested into **PostgreSQL + PostGIS** using `ogr2ogr`, enabling spatial joins and proximity queries that flat-file analysis cannot support. Coordinate system validation was applied at ingestion to catch mismatches before they could corrupt downstream analyses.

### Interactive Dashboard

A **Dash (Python)** application connects directly to the PostGIS database, exposing filters for neighborhood, time period, and data layer. The dashboard and database run in the same Docker network via Docker Compose, making the full stack reproducible from a single `docker compose up`—no manual setup, no environment drift.

## Key Analyses

- **Economic vulnerability**: Cross-tabulated income and employment data against neighborhood boundaries to surface high-need zones invisible in aggregated city-level statistics.
- **Service gap mapping**: Plotted health clinics, schools, and transit stops against population density to identify areas where infrastructure lags demand.
- **Environmental-health correlation**: Overlaid air quality sensor readings with respiratory illness rates by district, revealing spatial patterns not captured in either dataset alone.
- **Electoral geography**: Correlated voting behavior with socioeconomic indicators to understand what demographic forces shape local political outcomes in each neighborhood.

## Outcome

The platform delivers spatial queries in seconds that would otherwise require hours of manual GIS work. By unifying demographic, environmental, and electoral data in one interface, it gives planners a live view of how multiple forces intersect across the borough—making it possible to move from raw data to targeted policy proposals without data engineering overhead on every question.
