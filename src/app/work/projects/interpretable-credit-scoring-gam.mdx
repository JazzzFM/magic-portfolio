## Overview

[View on GitHub](https://github.com/JazzzFM/InterpretableRankingWithGeneralizedAdditiveModels)

In the financial sector, a credit model that can't explain its decisions isn't just a technical limitation—it's a regulatory one. GDPR's "right to explanation," CCPA, and local financial law all require that automated credit decisions be understandable by humans. This project delivers a complete MLOps system that achieves high predictive accuracy without sacrificing that interpretability.

## Why Generalized Additive Models

GAMs occupy a precise niche: they perform comparably to gradient-boosted trees on structured tabular data while remaining mathematically transparent. Each feature's contribution is modeled as an independent smooth function, making it possible to generate a chart showing exactly how each variable influenced a given score. This is what loan officers and regulators actually need—not a SHAP value attached to a black box, but a clear, auditable explanation of every decision.

## System Components

- **Model training**: Explainable Boosting Machine (EBM) via InterpretML, with automated hyperparameter search and cross-validation on historical lending data.
- **Ranking layer**: Learning-to-rank algorithms sort applicants not just by creditworthiness but by expected value—prioritizing the highest-return, lowest-risk applicants for the institution's portfolio.
- **Model serving**: FastAPI endpoint with sub-100ms inference and per-request explanation generation, returning both a score and a human-readable feature breakdown for every prediction.
- **MLOps pipeline**: Automated retraining triggers on new data, statistical drift detection compares live distributions against training baselines, and models promote through staging before reaching production.
- **Feature store**: PostgreSQL snapshots feature values at prediction time, making it possible to fully reproduce any past decision for audits or disputes—years after the fact.

## Outcome

The system closes the gap between ML performance and regulatory compliance—a gap that typically forces financial institutions to choose simpler, less accurate models. By using GAMs, this project demonstrates that interpretability and predictive power are not a trade-off but a design choice, enabling safer and more fair credit lending decisions without sacrificing business value.
