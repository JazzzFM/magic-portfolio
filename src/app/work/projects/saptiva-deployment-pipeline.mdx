## Overview

Deploying AI to banking, defense, and government institutions in Latin America means operating in environments built to resist change: air-gapped networks, procurement-locked hardware, multi-layer approval chains, and zero tolerance for unplanned downtime. At Saptiva AI, I built the deployment infrastructure that made this possible at scale—without bespoke engineering for every new client.

## The Core Problem

Every client had different constraints. Some ran fully on-premises with no internet access. Others required vendor certification before any software could be installed. All of them demanded full auditability of every change and a clear rollback path for every deployment. Building custom delivery processes per client was not viable beyond a handful of accounts. The solution was standardization.

## What I Built

- **Unified container-based delivery**: Packaged all Saptiva models and services as Docker images with pinned dependencies, enabling identical runtime behavior across wildly different host environments—from cloud VMs to isolated on-premises servers.
- **Air-gapped deployment runbooks**: Documented step-by-step procedures for environments with no internet access, covering offline image transfers, cryptographic signature verification, and rollback sequences. These runbooks passed security audits at national banking and defense institutions.
- **Model serving optimization**: Profiled and tuned the inference layer (FastAPI + quantized models) to meet the SLAs demanded by financial clients—typically sub-200ms for inference endpoints under production load.
- **Observability integration**: Deployed logging and alerting tailored to each client's existing SIEM infrastructure, enabling proactive detection of model drift and service degradation without requiring changes to the client's security stack.

## Technologies

Docker · Kubernetes · GitLab CI/CD · GitHub Actions · FastAPI · Go · Python

## Outcome

The standardized pipeline reduced prototype-to-production time by **40%**, enabling Saptiva to onboard clients faster while maintaining the security posture required by national defense and central banking entities. It also reduced the per-deployment engineering overhead significantly—freeing the team to focus on model quality rather than infrastructure coordination with each new client.
